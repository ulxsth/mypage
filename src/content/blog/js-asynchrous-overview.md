# 全体像
先にネタバレすると、JavaScript はこんな感じで解釈され、動いています。

(ここに GIF)

ブラウザとかで script タグが呼ばれたり、Node.js で書かれたコードが `node` コマンドで実行！されたとき、コードがイイ感じに分割され、イイ感じに順番に実行されます。
この実行される順番ですが、基本的には「上から、下に」ですね。これを **同期処理** といいます。
ただ、コードによっては「5秒経ってから処理してくれ！」とか、「メインの作業を止めないように他のとこで作業するで！」とかする場合が存在します。これが、俗にいうところの **非同期処理** なわけです。

この非同期処理は「あとまわしエリア」（仮）に突っ込まれ、メインの流れからいったん外されます。本当はもうちょっとカッコイイ名前がありますが、この記事ではそう呼びます。
そして、然るべきタイミングで処理されます。とりあえずはこれくらいの認識でいいと思います。

この一連の流れを、**イベントループ** といいます。
JavaScript の非同期処理について学ぶということは、イベントループについて知ることにほかなりません。そして、すべてを一度に理解しようとするととても難しいです。

こういうときは、最も身近な部分から攻めていくのがいいでしょう。
そのために、私たちが普段使っている、JavaScript における「非同期処理のための機能」とされているものを知るところから始めましょう。

# 非同期API
まずは、非同期APIとよばれるものです。
ここでいう "API" は、「JavaScript が提供してくれている、便利な機能」のことです。早い話、`console.log()` などがこれにあたります。
"Web API" とは別物なことに注意してください。

さて、ほとんどの関数は同期APIです。つまり、「呼び出したら、上から順に実行される」APIなわけですね。

```ts
console.log("1")
console.log("2")
console.log("3")

// 出力：
// 1
// 2
// 3
```

となると、非同期APIは「呼び出したら、特殊なタイミングで実行される」APIということになります。
`setTimeout()` は非同期APIのひとつで、指定された秒数のあとに、指定された関数を実行してくれる機能です。

```js
console.log("1")
setTimeout(() => console.log("2"), 1000)   // 1000ms (1秒) 後に実行
console.log("3")

// 出力：
// 1
// 3
// 2
```

このコードの面白いところは、`setTimeout` の行で1秒待っている間に、次の行の処理が行われてしまうことです。難しい表現を使うと、**ノンブロッキング** である、といえます。処理が終わるのを待たない、ブロッキングではない処理、ってことですね。

# async / await, Promise
JavaScript の非同期処理といえば、async/await ですね。

さて、「async/await とはなんですか？」と聞かれて、とっさに説明できますか？私はできません(この記事のために、理解を整理するまでは)。
なんとなくで使っている機能の代表、それが "async/await" です。

## async
まずは async について。
`async` は、関数の前につけるキーワードです。
これをつけると、その関数は「あとで終わる処理」を扱いやすい形になります。

たとえば、次のような処理はすぐには終わりません。

- API を叩いてデータを取ってくる
- ファイルを読む
- データベースに問い合わせる

こういう処理は、終わるまで待ち時間があります。
その待ち時間のあいだまで全部の処理を止めてしまうと不便なので、「終わったら結果を受け取る」という形で扱うことが多いです。
`async` は、そのための書き方だと思っておけばひとまず大丈夫です。

まずは普通の関数から見ていきます。
これは「こんにちは」を返すだけの関数。

```ts
function getGreeting() {
	return "こんにちは"
}

const result = getGreeting()
console.log(result)

// 出力:
// こんにちは
```

この関数に `async` をつけて、非同期にしてみます。
すると、`Promise` というよくわからないものが返ってくるようになりました。

```ts
async function getGreeting() {
	return "こんにちは"
}

const result = getGreeting()
console.log(result)

// 出力例:
// Promise { 'こんにちは' }
```

**Promise** とは、いったん「結果はあとで渡すよ」という引き換え券のようなものです。
実はこの Promise が、JavaScript の非同期処理をささえる概念になっています。

## Promise
Promise は「引き換え券」だと表現しました。これはどういうことでしょうか。
フードコートでご飯を頼むと、たいてい調理を待つ必要が出てきますね。そして、「調理が終わったらこれと引き換えてね」の印として呼び出しベルが渡されます。

![](https://i.imgur.com/raeEiAa.png)

Promise は、だいたいコレです。

実際に、Promise から値を受け取ってみます。
`Promise.then()` を使うと、Promise に紐づけられている処理が終わったあと、その結果を引数とした関数を呼び出すことができます。

```ts
async function getGreeting() {
	return "こんにちは"
}

const promise = getGreeting()

promise.then((value) => {
	console.log(value)
})

// 出力:
// こんにちは
```

ちょっとややこしいですね。それに、こんな書き方はあまりしません。
ここで、より簡潔に書く方法があります。そう、await ですね。

## await
さっきのコードは、これと同じです。[^1]

[^1]: 厳密にはasyncではない関数内やトップレベルでのawaitはできないんですが、ここではわかりやすさのために使っています。

```ts
async function getGreeting() {
	return "こんにちは"
}

console.log(await getGreeting())

// 出力:
// こんにちは
```

ここでは「async/await は、Promise を（内部的に）使ってかんたんに非同期処理するための書き方だよ」ということを把握しておけばよいかなと思います。

# イベントループと非同期処理
さて、この記事のミソの部分です。

さきほど、「非同期処理はいったん「あとまわしエリア」に投げ込まれる」と言いました。

# まとめ
- JavaScript の各処理のタイミングは **イベントループ** が管理してるよ
- 非同期処理はメインのループからはいったん外されて、イイ感じのタイミングで処理されるよ
- JavaScript の非同期処理には「非同期API」と「Promise（async/await）」があるよ

とりあえずこの辺がつたわればオッケーです。
実際には、後回しにされる処理にも種類があり、実行順にもルールがあります。このへんもそのうちかきます。
