# 非同期処理とは
そもそも、非同期処理ってなんなんだよ。
天下の MDN 様によると、以下のように定義されています。

> 非同期プログラミングは、長く続く可能性のあるタスクを開始しても、そのタスクが完了するまで待つのではなく、そのタスクの実行中も他のイベントに応答できるようにする技術です。タスクが完了すると、プログラムはその結果を表示します。

https://developer.mozilla.org/ja/docs/Learn_web_development/Extensions/Async_JS/Introducing

プログラムというのは、だいたい書いたとおりに上から実行されます。
非同期処理はこのルールにあてはまらない、同期処理とは違うタイミングで実行される処理のことを指します。

よくみるコードから、例をみてみます。
手っ取り早く非同期処理をしたければ、`async` をつけた関数を使うだけです。

```js
// 非同期な関数（中身は文字列を返すだけ）
const asyncFunc = async () => {
  return "The happiness is a warm pappy."
}
```

`async` をつけた関数は、そのまま呼び出すと `Promise` という型の帰り値を返します。こいつが JavaScript の非同期処理を支える概念になります。

```js
// 変数 promise には Promise 型のデータが返ってくる
const promise = asyncFunc()

console.log(promise)
// Promise { 'The happiness a worm pappy.' }
```

# Promise とは
ほな、その Promise ってのはなんやねん。
こいつは、一言でいえば「フードコートの受け渡し口」です。

あなたがフードコートに行って、バーキンでワッパーを頼んだとします。すると、ほとんどの場合は呼び出しベルを渡されますね。あなたはこれを持って適当なテーブルに腰掛け、こいつがなるまで呑気にTwitterを見たり、インスタを見たりします。その間にもキッチンでは調理が進み、ワッパーが完成したら呼び出しベルが鳴ります。そしたらあなたは受け渡し口に行って、ワッパーを受け取ってハッピーになります。
ここで、ワッパーを待つまでの呑気なネットサーフィンとワッパーの調理は非同期に行われています。ワッパーが作られているからといって、あなたのネットサーフィンが邪魔されることはありません。要するにこれが「非同期処理」です。そして、非同期に作られたワッパーはあなたに受け取られる必要がありますね。このとき、お店とあなたの間にあるのは「受け渡し口」です。こいつが Promise です。「ワッパーを買う」という「約束」があったので、最終的にここでワッパーが受け取れたわけですね。
